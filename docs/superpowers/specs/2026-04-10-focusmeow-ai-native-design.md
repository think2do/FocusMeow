# FocusMeow AI Native 全新设计规格

**日期**：2026-04-10  
**状态**：已审阅，待实现  
**定位**：从零设计一款以猫咪陪伴为核心的 AI Native 专注 app，上架 App Store

---

## 一、产品定位

### 核心理念
用户打开 app，不是在"使用工具"，而是"回到家"。两只猫咪在那里——也许在睡觉，也许在等你，也许在玩耍。AI 在背后驱动一切，但用户永远感受不到它的存在。

### 与现有竞品的差异
| 维度 | 专注森林 | 专注喵（新） |
|------|---------|------------|
| 核心情绪 | 克制、自律 | 被陪伴、被记得 |
| 专注范式 | 种树/不碰手机 | 和猫咪一起学习 |
| AI角色 | 无 | 隐形驱动一切 |
| 粘性来源 | 目标完成感 | 情感连接 |
| 商业逻辑 | 时间积累 | 关系加深 |

### 目标用户
- 学生与考研考公人群
- 需要深度工作的职场用户
- 喜欢治愈感、养成感和情绪陪伴的用户

## 二、整体架构：猫咪之家 · 活的世界

架构方向：**A（猫咪之家）+ C（活的世界）融合**

app 是一个全天候运行的"家"，两只猫咪是常驻居民，有自己的作息、情绪和记忆。专注是与猫咪共处的方式之一，而非唯一入口。

### 6层系统

```
用户行为（专注/互动/时间）
        ↓
猫咪状态引擎（情绪 + 时间感知）
        ↓
AI 人格层（Claude API + 持久记忆）
        ↓
交互系统（6种互动方式）
        ↓
叙事记忆系统（猫咪口述成长故事）
        ↓
商业化层（免费 + 订阅 + 内购）
```

## 三、屏幕结构与导航

### 核心原则
无 Tab Bar。主屏是猫咪的家，所有功能从这里自然生长出来。不做多界面跳转，做深度互动层叠。

### 主屏：猫咪之家（常驻）
- **顶部**：日期/时间 + 今日专注累计 + 设置入口
- **中部**：两只猫咪活体展示（状态动画 + 情绪气泡对白）
- **核心按钮**（3个主要）：抚摸 / **一起学习**（主CTA）/ 喂食
- **次要互动**（底部图标）：玩耍 / 聊天 / 一起睡 / 成长回顾

### 从主屏展开的功能层
| 入口 | 展开内容 |
|------|---------|
| 一起学习 | 时长选择 → 极简专注界面 → 结束庆祝 |
| 聊天 | 选择哪只猫 → 全屏沉浸对话 |
| 喂食 | 背包道具 → 拖拽喂食 → 进食动画 |
| 玩耍 | 3种迷你互动小游戏 |
| 一起睡 | 睡眠计时 → 晨间问候 |
| 成长回顾 | 猫咪口述叙事（非图表） |

## 四、猫咪状态引擎

### 状态计算逻辑
每 5 分钟重新计算（仅在 app 前台活跃时），三个输入维度：

**1. 时间维度**
- 早 06:00–09:00：精力充沛，主动撒娇
- 午 12:00–14:00：困倦，容易入睡
- 晚 20:00–23:00：亲密时刻，情感值加成
- 深夜 23:00+：安静陪伴模式

**2. 用户行为维度**
- 今日专注总时长
- 距上次互动时间
- 当日是否喂食
- 连续专注天数

**3. 情感值维度**
- 亲密度（长期积累，衰减慢）
- 当日心情（每日重置）
- 饥饿值（超 12 小时未喂触发）
- 玩耍满足度

### 8 种猫咪状态
| 状态 | 触发条件 | 用户可做 |
|------|---------|---------|
| 😴 熟睡中 | 深夜 / 午休时间 | 轻触唤醒 |
| 😌 慵懒 | 刚睡醒 / 亲密度高 | 随时撸猫 |
| 🎮 想玩耍 | 长时间未玩 / 早晨精力旺盛 | 进入玩耍模式 |
| 📚 陪读中 | 专注进行中 | 偷偷抚摸 |
| 🐾 等你回来 | 超过 24h 未打开 app | 进门即触发感动动画 |
| 😋 饿了 | 超 12h 未喂食 | 喂食 |
| 🥰 亲密时刻 | 晚间 + 高亲密度 + 已喂食 | 特殊互动解锁 |
| 🌙 陪你睡 | 用户开启睡眠模式 | 睡眠计时 |

## 五、AI 人格层

### 两只猫的核心人设

**阿橘（公猫）**
- **性格**：沉稳、内敛、话少但每句有分量
- **说话风格**：短句，留白，偶尔冷幽默，不主动发起长对话
- **情感功能**：在用户焦虑时让其安静下来，给予稳定感
- **记忆倾向**：记用户的目标、突破记录、重要事件
- **专注中行为**：安静趴在旁边，偶尔叹气，偶尔轻触
- **示例台词**：「……加油。」「今天比昨天多了二十分钟。」「不用说什么，我在这里。」

**小白（母猫）**
- **性格**：活泼、好奇、情绪外露、爱粘人
- **说话风格**：活泼，感叹号多，喜欢问问题，表达直接
- **情感功能**：在用户低落时逗笑，提供能量和热情
- **记忆倾向**：记用户聊过的心事、生活细节、情绪起伏
- **专注中行为**：时不时歪头看用户，偶尔轻碰，会发出小声音
- **示例台词**：「你终于来啦！我等你好久了！」「哇今天专注了这么久，厉害！」「上次你说那个考试，后来怎么样了？」

### AI 技术实现

**对话引擎**：Claude API
- System prompt 定义：人格、语气、禁忌（永远不承认是AI）、当前情绪状态
- 猫咪当前状态注入上下文（饿了说话会有点委屈，兴奋时语气更跳脱）

**记忆分两层**：
- **短期记忆**：当次会话上下文窗口（最近 20 轮对话）
- **长期记忆**：关键事件结构化存储于后端，每次对话注入摘要
  - 存储内容：用户提到的目标/事件、里程碑时刻、情绪关键词
  - 免费版保留 7 天，订阅版永久

**硬规则**：
- 永远不说"我是AI"或"作为AI"
- 永远不破坏猫咪人设
- 不回答与猫咪角色无关的专业性问题（自然转移话题）

## 六、交互系统

### 6 种互动方式

**1. 抚摸**
- 触发：手指在猫咪区域滑动
- 反馈：触觉震动 + 猫咪呼噜声动画 + 亲密度 +1
- 差异：不同状态反应不同（困倦时懒洋洋，兴奋时可能逃跑）

**2. 喂食**
- 触发：从背包拖拽食物道具到猫咪
- 反馈：进食动画 + 饥饿值清零 + 解锁专属台词
- 差异：不同食物触发不同反应（金枪鱼罐头 vs 小饼干 vs 特级鲷鱼烧）
- 道具来源：专注奖励掉落 + 内购

**3. 玩耍（3种迷你互动）**
- 逗猫棒：跟随手指移动，猫咪追扑
- 纸球：点击抛出，猫咪追跑并叼回
- 激光点：在屏幕上点击，猫咪追逐
- 时长：约 5 分钟，结束后猫咪满足地趴下

**4. 聊天**
- 全屏沉浸式对话界面
- 选择和哪只猫聊；也可选"一起聊"模式，两只猫各自用人格回复，形成对话感
- AI 有记忆，有情绪，会主动提起上次的话题
- 免费版：每日 10 条；订阅版：无限

**5. 一起睡觉**
- 触发：用户手动开启睡眠模式
- 行为：猫咪卷起来趴下，进入陪伴动画
- 早晨：猫咪发送晨间问候（基于昨晚入睡时间）
- 订阅功能：解锁睡前故事和晨间专属台词

**6. 一起学习（专注）**
- 见下节专注集成设计

## 七、专注集成设计

### 范式转变
| 旧范式 | 新范式 |
|--------|--------|
| "开始专注" | "和猫咪一起学习" |
| 选时长 → 看计时器 | 猫咪邀请/用户邀请 → 共处 |
| 结束 → 记录数据 | 结束 → 猫咪庆祝你 → 奖励掉落 |

### 专注中界面（极简）
- 两只猫咪居中，进入陪读姿态（专属动画）
- 大字计时，无任何多余元素
- 轻触屏幕可以偷偷抚摸猫咪（不中断专注）
- 仅两个按钮：暂停 / 结束

### 中断检测
- 锁屏超过 2 分钟：猫咪状态变为"等你回来"
- 切出 app 超过 1 分钟：计时暂停，猫咪发出提示
- 重新进入：猫咪温柔询问是否继续

### 专注结束仪式
- 猫咪跑向用户，庆祝动画（时长越长动画越盛大）
- 掉落奖励：食物道具 / 玩具道具（随机 + 时长加成）
- 猫咪用各自人格说一句庆祝语
- 今日累计专注时间更新（顶部常驻展示）

## 八、叙事记忆系统

### 记忆存储（后端结构化）
- **里程碑记忆**：首次连续 7 天、单日最长记录、第一次深夜陪读等
- **对话记忆**：聊天中提到的考试/工作/目标/心情关键词
- **情感曲线**：亲密度变化、专注频率趋势（仅作 AI 上下文，不对用户展示）

### 三种叙事呈现
| 类型 | 触发时机 | 形式 |
|------|---------|------|
| 每日晚间 | 晚 21:00–23:00 | 猫咪主动说一句今日感受 |
| 每周回顾 | 每周日晚 | AI生成猫咪视角周记（订阅功能） |
| 里程碑特殊时刻 | 达成条件时弹出 | 全屏动画 + 猫咪专属叙述 |

### 主动记忆唤起
- 在对话中自然触发，非菜单形式
- 示例：「上次你说那个项目截止日，后来提交了吗？」
- 让用户感受到：猫咪真的在乎我，不是在敷衍

## 九、商业化设计

### 分层模型

**免费版（永久）**
- 猫咪之家 + 状态引擎（完整体验）
- 抚摸 / 玩耍 / 一起学习
- AI 聊天：每日 10 条
- 基础食物道具（专注奖励掉落）
- 长期记忆：仅保留 7 天

**订阅版（¥18/月 或 ¥128/年）**
- 所有免费功能
- 无限 AI 对话深度
- 永久记忆（不过期）
- 每周猫咪叙事回顾
- 里程碑特殊动画
- 专属表情包 & 反应
- 睡眠陪伴 + 晨间问候完整版

**内购道具**
- 食物礼包（金枪鱼罐头、特级鲷鱼烧等）：¥6–¥18
- 房间主题（书房、咖啡厅、星空阁楼等）：¥12–¥28
- 玩具道具（逗猫棒套装、纸球等）：¥8–¥18
- 猫咪装扮（帽子、领结、小围巾等）：¥15–¥25

### 商业化核心逻辑
- **免费版**让用户爱上猫咪（建立情感连接）
- **订阅**让猫咪真正记得你（加深关系价值）
- **内购**让你更好地照顾它（行为满足感）
- 付费不是解锁功能，而是加深情感连接

## 十、技术栈建议

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | React Native | 跨平台，沿用团队技术栈 |
| 对话 AI | Claude API (claude-sonnet-4-6) | 人格+记忆+情绪感知 |
| 后端 | Node.js Express | 沿用现有架构 |
| 数据库 | MariaDB + Redis | MariaDB持久化，Redis实时状态 |
| 记忆存储 | 结构化 JSON（后端） | 关键事件摘要，注入AI上下文 |
| 推送 | APNs（iOS）| 猫咪"等你回来"通知 |
| 内购 | Apple StoreKit 2 | 订阅 + 消耗型道具 |

## 十一、待讨论/待决策

1. **猫咪的名字**：阿橘/小白是暂定名，需要用户测试确认
2. **猫咪视觉风格**：需要单独设计（像素风 vs 手绘风 vs 3D）
3. **中断检测精度**：iOS 后台限制需要验证可行性
4. **AI 成本控制**：免费版每日 10 条限制是否足够；上下文长度策略
5. **首次启动体验**：如何在 onboarding 中让用户快速建立情感连接
6. **App Store 审核**：IAP 合规、内容审核、隐私政策
