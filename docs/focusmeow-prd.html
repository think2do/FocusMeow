Core Framework

产品核心框架

FocusMeow 的底层不是单一功能结构，而是四层叠加的产品骨架。

01

专注工具层

承载最基础的任务执行与时间管理能力，保证产品在效率维度可用。

  • 选择陪伴猫咪
  • 选择或自定义专注时长
  • 倒计时与结果判定
  • 中断、离开前台的行为识别
02

游戏养成层

把抽象的专注转成具体的奖励、升级、收集和惩罚，构成持续驱动力。

  • 专注完成获得 XP
  • 每 2 次专注掉落新猫
  • 时长越高，稀有猫概率越高
  • 连续中断会造成挨饿与死亡
03

情感陪伴层

让用户和“自己的猫”形成情绪连接，降低传统效率产品的机械感。

  • 首页猫咪王国
  • 猫咪展示位与主题切换
  • AI 谈心
  • 单猫独立聊天记忆
04

智能解释层

把数据结果翻译成用户能理解的结论和建议，形成“教练式反馈”。

  • 最佳专注时段
  • 最佳专注时长
  • 趋势对比
  • 连续专注与习惯建议
User Loop

核心用户闭环

这是产品最重要的路径：行为输入、情绪反馈、成长沉淀和再次启动。

入口

进入首页

用户看到自己的猫咪王国、累计数据和距离下一只猫的进度。

决策

选择猫 + 时长

从“我要开始做事”变成“我要带一只猫一起开始专注”。

执行

专注中

猫咪在进食，离开会被警告，放弃会让用户感受到情绪成本。

反馈

成功 / 中断 / 死亡

即时看到鼓励、XP、掉落结果，或挨饿与死亡反馈。

沉淀

图鉴 / 记录 / AI

用户回看成长、收集进度，并在 AI 页得到对行为的解释。

回流

再次启动专注

通过养成目标和陪伴关系，推动用户继续下一轮专注行为。

Modules

功能架构 PRD

以下按正式 PRD 方式描述各模块目标、作用和关键业务规则。

01. 账号系统

登录 / 注册 / 游客

负责身份进入、登录态恢复与低门槛试用，是转化闭环的前置模块。

核心功能

  • 欢迎页、登录、注册、验证码、忘记密码、重置密码
  • 游客模式直接进入
  • 登录态本地持久化
注册需邮箱 + 密码 + 昵称
邮箱验证码验证注册
游客可体验但能力受限

02. 首页 / 猫咪王国

主入口

负责品牌感、资产展示和任务启动，是用户每天回到产品时最先看到的“世界”。

核心功能

  • 品牌头部、累计数据、猫咪王国展示位
  • 主题切换、规则帮助、截图分享
  • 开始专注主 CTA
首页最多展示 6 个猫咪位
支持固定展示喜欢的猫
主题按拥有猫数解锁

03. 专注系统

行为引擎

负责专注会话的启动、执行、倒计时和结果判定，是所有成长数据的源头。

核心功能

  • 选择猫咪、时长预设、自定义时长
  • 倒计时页面、专注态沉浸展示
  • 放弃确认、前后台切换检测
  • 成功 / 中断 / 死亡结果页
未选猫不可开始专注
完成后写入 focusHistory
中断会触发负反馈

04. 猫咪养成系统

长期动力

把专注行为转译为成长资产，让每次专注都有长期可累计结果。

核心功能

  • 初始猫、XP、等级、存活状态
  • 死亡状态、复活机制、陪伴时长
  • 稀有与普通分类
成功专注给当前猫加 XP
XP 达阈值后升级
初始猫具备保底复活

05. 猫咪掉落与图鉴

收集系统

通过短周期奖励维持动力，并用图鉴作为长期目标板。

核心功能

  • 每 2 次专注获得新猫
  • 普通猫 / 稀有猫随机掉落
  • 普通图鉴与稀有图鉴
  • 收集进度展示
累计时长越高，稀有概率越高
新品种自动点亮图鉴
未解锁品种以隐藏状态展示

06. 成长记录系统

资产回看

帮助用户明确感知自己的总投入、单猫成长和当前养成规模。

核心功能

  • 总完成次数、总专注时长、猫咪总数、最高等级
  • 全部猫咪列表、经验条、状态和陪伴时长
  • 死亡猫保留展示，支持部分丢弃
初始猫不可丢弃
死亡猫保留历史痕迹

07. AI 谈心系统

情感陪伴

增强猫咪人格感，让用户能与不同猫建立关系，而不是只和系统聊天。

核心功能

  • 按猫切换对话对象
  • 单猫聊天历史与上下文
  • 历史保存、新建对话、主题摘要
  • 消息复制 / 撤回
历史保存上限为 5 只猫
超过上限需替换
游客聊天次数有限

08. AI 分析系统

教练反馈

基于专注历史生成最佳时段、最佳时长、趋势和建议，增加“科学感”。

核心功能

  • 今日 / 本周 / 本月报告
  • 完成率、分钟数、趋势对比
  • 最佳时段、最佳时长、连续专注天数
  • 建议模块
当前以本地规则分析为主
重点是可解释反馈而非长文本生成

09. 设置与资料系统

账号与偏好

支持头像、昵称、个人信息、语言和反馈配置，补足用户身份感和可管理性。

核心功能

  • 头像选择 / 拍照 / 相册上传
  • 昵称、邮箱、密码管理入口
  • 生日、性别、身份、兴趣等个人信息
  • 语言、反馈、条款、隐私、版本
个人信息用于增强“猫更懂你”的感觉
设置承担注册转化和账号管理职责
User Stories

关键用户故事

以下故事决定了 FocusMeow 是否能成立，而不是是否“功能齐全”。

新用户

我想低门槛试用

作为第一次接触产品的用户，我希望不注册也能快速开始体验，以便判断这款产品是否值得长期使用。

  • 需要游客模式
  • 需要保留核心体验
  • 需要通过限制推动后续注册
专注用户

我想更容易坚持

作为想培养专注习惯的用户，我希望完成一次专注后能获得明确回报，而不是只是看到一个结束提示。

  • 需要即时正反馈
  • 需要短周期成长奖励
  • 需要失败时有感知成本
养成用户

我想积累自己的猫咪资产

作为喜欢收集和养成的用户，我希望看到自己拥有的猫越来越多、越来越强，并能展示它们。

  • 需要图鉴和稀有体系
  • 需要首页王国展示
  • 需要等级和陪伴时长沉淀
情感用户

我希望和猫建立关系

作为偏情绪型使用者，我希望我的猫不只是奖励素材，而是能和我持续交流、陪伴我的角色。

  • 需要 AI 谈心
  • 需要按猫切换对话对象
  • 需要独立聊天记录
反思用户

我想知道如何更高效

作为有提升意愿的用户，我希望知道自己在什么时段更高效、适合什么时长，以及最近有没有进步。

  • 需要时段分析
  • 需要时长分析
  • 需要趋势和建议
产品目标

我希望形成长期留存

作为产品本身，FocusMeow 需要让用户因为“还有猫没收集、还有王国想经营、还有数据想变好”而持续回来。

  • 用专注驱动掉落
  • 用掉落驱动回访
  • 用 AI 和陪伴延长生命周期
Key Rules

关键业务规则

这些规则共同定义了产品的激励与惩罚逻辑，是玩法成立的基础。

专注规则

  • 用户需先选择一只猫后才能开始专注。
  • 专注完成会记录完成状态、日期、起始小时、时长和计划时长。
  • 专注中主动放弃或离开前台会触发负反馈。

成长规则

  • 成功专注后，当前陪伴猫获得固定 XP。
  • XP 达到阈值后升级，等级随累计 XP 增长。
  • 每累计 2 次成功专注会掉落 1 只新猫。

稀有掉落规则

  • 两次专注的累计时长影响稀有猫概率。
  • 时长越高，稀有掉落概率越高。
  • 稀有类型当前包括情侣猫、彩虹猫、黑猫。

惩罚规则

  • 中断专注会让猫挨饿。
  • 连续中断达到阈值会导致当前猫死亡。
  • 初始猫拥有保底复活机制，降低早期挫败感。

聊天规则

  • 每只猫可作为独立聊天对象。
  • 聊天历史保存数量有上限，超过上限需替换。
  • 聊天轮次达到一定程度后可生成主题摘要。

游客规则

  • 游客可体验核心功能，但专注、聊天、AI 分析次数受限。
  • 游客模式承担“试用”而非“长期资产沉淀”职责。
  • 正式注册用于承接长期记录与留存。
Metrics

核心指标建议

如果后续做更正式的产品运营，这些指标最能反映产品是否真正跑起来。

使用

基础活跃与完成

DAU / WAU 判断产品是否形成稳定回访。
人均专注次数 判断产品是否被真正用于执行专注。
专注完成率 判断玩法设计是否帮助用户坚持而不是劝退。
养成

成长系统健康度

人均拥有猫咪数 判断用户是否持续进入掉落循环。
图鉴点亮率 判断收集目标是否有持续吸引力。
主题解锁率 判断猫咪资产是否进一步转成展示欲望。
AI

情感与解释层使用度

聊天页打开率 判断猫咪人格化是否成立。
人均消息数 判断 AI 陪伴是否形成真实互动。
分析页回访率 判断 AI 建议是否被视为有价值。
